# Security Policy

We are committed to keeping our tools and platforms safe.

This repository defines the organization-wide security reporting process.
Individual repositories may provide additional details where necessary.

## Reporting a Vulnerability

**Please do not open public issues for security reports.**
If you discover a vulnerability, report it privately using one of the following methods:

  - **GitHub Private Reporting:** Use the "Report a vulnerability" button under the Security tab of the affected repository.
  - **Email:** <!-- email_address -->

Please include:

  - The affected repository and version
  - A clear description of the vulnerability
  - Steps to reproduce the issue
  - Potential impact or exploit scenarios
  - Any relevant logs, screenshots, or proof of concept

What Happens Next?

  - **Acknowledgment** – We will acknowledge receipt of your report as soon as reasonably possible.
  - **Investigation** – We will review, validate, and assess the impact.
  - **Resolution** – We will work on a fix or mitigation.
  - **Disclosure** – Once resolved, we will coordinate responsible disclosure.
