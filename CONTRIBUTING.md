# Contributing

Thank you for your interest in contributing to NeatPlatform!
This document is intentionally brief; individual repositories may have more specific contribution guidelines.

## Issues

  - Search for existing issues before opening a new one.
  - Use the appropriate issue template when available.
  - Use clear, descriptive titles and provide steps to reproduce bugs or detailed feature requests.

## Pull Requests

  - Fork the repository and create a feature branch off `main`.
  - Make small, focused changes and include a description of what the PR does.
  - Reference any related issues using `#<issue-number>`.
  - All commits must be signed to establish trust.
  - Consider using [Conventional Commits](https://www.conventionalcommits.org) for commit messages.
  - Ensure that any added code is properly formatted and passes existing tests.
  - Include new or updated tests for your changes and make sure coverage criteria are met.

## Communication

  - Be respectful and constructive in all interactions. Always assume positive intent.
  - Consider using the [Conventional Comments](https://conventionalcomments.org) format when reviewing or giving feedback.
  - If you are unsure how to proceed, feel free to open an issue and ask for guidance.

## Code of Conduct

  - All contributors are expected to follow the [NeatPlatform Code of Conduct](CODE_OF_CONDUCT.md).
